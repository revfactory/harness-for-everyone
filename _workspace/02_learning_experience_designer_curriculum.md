## 0. 설계 원칙 (팀원 열람용 한 장 요약)

1. **이해는 순서다.** 선행 개념을 건너뛰면 뒤 개념이 무너진다. 모든 레슨은 위상 정렬된 그래프를 따른다.
2. **한 레슨 = 새 용어 3개 이하.** 초과 시 반드시 분할.
3. **도달 목표는 관찰 가능한 행동으로.** "이해했다" 대신 "세 문장으로 말할 수 있다".
4. **오개념을 예상하고 먼저 짚는다.** 청자가 틀리기 쉬운 지점을 레슨 중간에 방어적으로 언급.
5. **심리 장벽을 낮춘다.** "못 따라갈 것"이라는 신념이 들면 이해는 시작되지 않는다. "이해만 해도 충분하다"는 메시지를 반복한다.
6. **은유는 한 단위 안에서 섞지 않는다.** 주방을 택한 레슨은 끝까지 주방으로 간다.
7. **동심원형 자립성과 위상 정렬의 공존 규칙 (R-LXD-11 반영).** Lv.2/Lv.3은 Lv.1 미이수자도 시작할 수 있도록 **"5분 프리레슨"**을 레벨 초입에 압축 배치한다. 프리레슨은 해당 레벨에서 가정하는 선행 개념(P1~P4 중 필요분)을 주방 은유로 환기만 하고 넘어간다. 이로써 "자립성(각 레벨 독립 진입 가능)"과 "위상 정렬(선행 없이는 뒷 개념이 무너짐)" 두 원칙이 양립한다.
8. **도입부 주어 명시 원칙 (Rev.3 · concept-translator v2 반영).** 하네스를 처음 정의하는 모든 문장은 **주어 "사람이 / 우리가 짠다"**를 반드시 포함한다. "하네스는 여러 AI가 일하게 만드는 매뉴얼이다"가 아니라 **"하네스는 사람이 짜는, 여러 AI가 역할을 나눠 일하게 만드는 운영 매뉴얼이다"**로 시작한다. 이 한 줄의 차이가 M3(제품 오인)·M7(사람 대체 오인)·M1(AI 의지 오인)을 **첫 인상에서 선제 방어**한다. 1분 영상·3컷 카드뉴스·Lv.1 레슨의 첫 문장은 이 규칙을 위반할 수 없다.

## 1. 개념 지도 (Concept Map · 위계 그래프)

하네스 이해를 위해 반드시 통과해야 하는 개념의 위계. **위에서 아래로 선행 → 본 → 파생** 순으로 읽는다. 들여쓰기는 의존성(= "이 개념을 이해하려면 윗단을 알아야 한다")을 뜻한다.

```
[선행 전제 계층 · Prerequisite Layer]
└─ P1. "AI는 글·코드·표를 생성하는 도구다" (챗봇 경험 수준)
   └─ P2. "같은 AI라도 어떻게 부탁하느냐에 따라 결과가 다르다" (프롬프트 감각)
      └─ P3. "사람은 팀을 꾸려 분업하면 혼자보다 잘한다" (협업의 일상 직관)
         └─ P4. "AI는 한 번에 기억할 수 있는 양이 정해져 있다" (한계 인식)

[핵심 개념 계층 · Core Layer]
└─ C1. 에이전트 (Agent) — "역할을 맡은 한 명의 AI 셰프"
   └─ C2. 스킬 (Skill) — "에이전트가 꺼내 보는 레시피 카드"
      └─ C3. 오케스트레이터 (Orchestrator) — "헤드셰프, 누구에게 뭘 시킬지 결정"
         └─ C4. 하네스 (Harness) — "사람이 매일 고쳐 쓰는 주방 운영 매뉴얼" [본 개념]

[파생·확장 계층 · Extension Layer]
├─ E1. 서브에이전트 (Sub-agent) — "그 한 접시만 맡고 떠나는 게스트 셰프"
├─ E2. 컨텍스트 윈도우 (Context Window) — "조리대 위 공간의 한계"
├─ E3. 메모리 (Memory) — "영업 끝나면 적어두는 조리 노트"
├─ E4. 도구·MCP (Tool / MCP) — "주방 장비 + USB-C 같은 표준 연결 포트"
└─ E5. 하네스 팀 (Harness Team) — "한 영업을 함께 돌리는 주방 전체"

[오개념 방어 계층 · Correction Layer]
└─ X. "AI는 사람이 아니다 — 기억 휘발·순간 복제·평등 협업·도구 오남용"
    (이 계층은 모든 레슨에 스며든다. 단독 레슨 아님.)
```

### 위상 정렬 체크리스트

- C1(에이전트) 이전에 P1·P2가 통과되었는가? → 안 되면 학습자는 "그냥 AI랑 뭐가 다른데?"에서 멈춘다.
- C3(오케스트레이터) 이전에 C1·C2가 통과되었는가? → 안 되면 "혼자 다 하는 똑똑한 AI"로 오해한다.
- C4(하네스) 이전에 C1·C2·C3이 통과되었는가? → 안 되면 "AI 제품 이름"으로 오해한다.
- E1~E5는 C4 이후에만 도입한다. 순서를 바꾸면 학습 부하가 급격히 올라간다.

### 우회 경로 vs 정면 돌파 경로

**정면 돌파 (권장):** P1 → P2 → P3 → P4 → C1 → C2 → C3 → C4 → E1~E5 → X

> ⚠️ **우회 경로의 비용 명시 (R-LXD-04 반영).** 선행 개념을 생략하면 일부 오개념은 **교정 불가**가 된다. 우회 경로는 "더 쉬운 학습"이 아니라 "더 **좁은** 학습"이다. 도달 목표도 축소되어야 한다.

**우회 (디지털 낯선 중장년용):** P1 → P3 → C1 → C3 → C4 → (E3 선택적) → X 부분 적용
- P2·P4 생략. "역할 분업" 일상 직관만으로 본 개념까지 진입.
- **축소된 도달 목표:** 하네스가 무엇인지 한 문장으로 말할 수 있다 + 오개념 **M3(제품 아님)·M7(사람 대체 아님)** 두 가지만 교정한다.
- **교정 포기 오개념 (명시):** M9(프롬프트 = 검색어)·M10(컨텍스트 무한 기억)은 P2·P4가 없으면 교정 불가. 이 경로의 청중에게는 해당 오개념 주제 자체를 언급하지 않는다. 대신 "AI에게 부탁할 때는 맥락을 길게 적으면 좋다"는 실천 수칙 1개만 전달한다.
- 파생 계층은 E3(조리 노트)만 다룬다 — E1·E2·E4는 생략.

**우회 (초중고생용):** P1 → P3 → C1 → C2 → C3 → C4 → E4(도구만) → X 부분 적용
- P2·P4 생략, 스포츠팀 전술판 은유로 빠르게 진입 후 주방 은유로 환승 가능 (단, 환승은 섹션 경계에서만).
- **축소된 도달 목표:** 하네스의 핵심 3요소(에이전트/스킬/오케스트레이터)를 말할 수 있다 + 오개념 **M2(에이전트 = 사람 직원)·M13(에이전트 = 챗봇)** 두 가지만 교정한다.
- **교정 포기 오개념 (명시):** M9·M10은 정면 돌파 경로를 원하는 학생에게만 별도 제공.

## 2. 3단 학습 경로 (Learning Path)

학습 경로는 **학습자의 투입 시간**을 기준으로 입문(Lv.1) / 중급(Lv.2) / 심화(Lv.3)로 설계한다. 각 레벨은 **이전 레벨 없이 시작해도 최소 이해가 성립**하도록 자립 가능하게 설계한다 (누적형이 아닌 동심원형).

### 📘 Lv.1 — 입문 경로 (1분 · "하네스가 뭔지만 말할 수 있게")

**투입 시간:** 1~10분 (1분 영상 / 3분 카드뉴스 / 10분 읽기 중 택1)

**도달 목표 (Learning Objective):**
- 학습 종료 시, 학습자는 비전문가 친구에게 하네스를 **한 문장**으로 설명하고, **자기 일상에서 유사한 예를 하나 들 수 있다**.
- Lv.1은 **"존재를 알고 일상에 빗댈 수 있다"**까지만 목표로 한다. 제품/설계 구분, 챗봇/에이전트 구분 등 변별 과제는 Lv.2로 이관. (R-LXD-03 반영)

**완료 기준 (Done Criteria) — 관찰 가능한 행동:**
1. **(전이 Transfer — 필수)** 자기 주변 일상 상황을 하나 골라 "이건 하네스적이다 / 아니다"라고 판별하고 한 문장으로 이유를 말할 수 있다.
   - *통과 예시:* "김장할 때 엄마가 배추 담당·양념 담당·포장 담당 나누는 것과 비슷해. 역할을 나눠 함께 하니까."
   - *미통과 예시 (단순 재구성):* "AI들이 함께 일하는 거야" — 전이 증거 없음.
2. **(구분 Discrimination — 보조)** "에이전트"와 "AI" 중 어느 쪽이 더 좁은 개념인지 답할 수 있다 (에이전트 = 역할 부여된 AI).

> **완료 기준 설계 근거 (R-LXD-01 반영).** Bloom's taxonomy의 "이해(Understand)"는 paraphrase만으로 충족되지 않으며 transfer(전이)·discrimination(구분)을 요구한다. Rev.1은 paraphrase 단독으로 통과 가능해 "기억력 측정"에 머물렀다는 지적을 수용, 완료 기준 #1을 전이 기반으로 재설계했다.
>
> **미해결 리스크 (잔존).** 현장 검증 전까지 "청중 50% 이상이 일상 예를 들지 못할 가능성"은 잔존한다. 이 리스크는 synthesis-lead가 최종 제안서 "미해결 리스크" 섹션에 명기한다.

**레슨 구성 (4개 · R-LXD-03 반영):**

| 순서 | 레슨명 | 분량 | 다루는 개념 (3개 제한) | 핵심 은유 |
|---|---|---|---|---|
| L1-1 | "왜 AI 하나로는 부족한가" | 25초 | P1, P3 | 주방에 요리사 한 명만 있을 때 |
| L1-2 | "역할을 나누면 달라진다" | 25초 | C1, C3 | 셰프·헤드셰프 |
| L1-3 | "이 전체를 묶어 **사람이 짜는** 하네스" | 25초 | C4 | 주방 운영 매뉴얼 (매일 고쳐 쓰는) |
| L1-4 | **"내 일상에서 찾아보기 (전이 프롬프트)"** | 25초 | — (전이) | "오늘 점심 먹은 식당도 그랬을까?" 질문형 마무리 |

- Rev.1의 "제품 아님" 교정은 Lv.2로 이관했다. Lv.1의 90초에는 오개념 교정을 위한 시간이 부족하다는 지적(R-LXD-03) 수용.

**사용 전달 포맷:** 1분 영상 스크립트(concept-translator 5.1 활용, 마지막 5초에 "여러분의 일상에서 비슷한 상황을 하나 떠올려보세요" 질문 삽입) / 4컷 카드뉴스 / 350자 글

### 📗 Lv.2 — 중급 경로 (30분 · "하네스가 어떻게 돌아가는지 그릴 수 있게")

**투입 시간:** 20~40분 (3분 영상 + 10분 글 + Q&A 5분 + 자가점검 5분 권장 조합)

**도달 목표:**
- 학습자는 하네스의 다섯 요소(에이전트/스킬/오케스트레이터/도구·MCP/메모리)를 **각각 한 문장**으로 설명하고, 다섯 요소 사이의 흐름을 **순서도로 그릴 수 있다**.
- 학습자는 "AI가 알아서 다 해준다"는 오개념과 "하네스는 제품이다"라는 오개념 2개를 교정할 수 있다.

**완료 기준:**
1. 주방 비유로 하네스의 다섯 요소를 각각 한 문장으로 설명할 수 있다.
2. "프롬프트 → 오케스트레이터 → 에이전트들 → 결과 통합 → 사용자"의 흐름을 순서대로 말할 수 있다.
3. "내가 오늘 AI와 한 작업이 하네스적인가?"를 스스로 판별하고 그 이유를 설명할 수 있다.
4. "AI가 스스로 판단한다"라는 문장의 참과 거짓을 구분해 설명할 수 있다 ("판단의 범위가 설계된 안에서만").

**레슨 구성 (8개 · R-LXD-02 반영):**

L2-6a(공간·기록 2개)와 L2-6b(실패지점 4개를 **한 묶음 원칙**으로 제시 — 개별 용어가 아닌 "사람-AI 차이의 4가지 규칙"이라는 하나의 프레임)로 분할해 각 레슨의 신규 인지 부하를 3개 이하로 낮췄다.

**사용 전달 포맷:** 3분 영상(concept-translator 5.2 씬 단위) + 10분 블로그 글 + 자가점검 질문지

### 📕 Lv.3 — 심화 경로 (1시간 · "하네스를 타인에게 설명할 수 있게")

**완료 기준:**
1. 주방 은유를 사용하여 10분 분량의 설명을 구조화해 말할 수 있다 (도입 훅 → 개념 1:1 매핑 → 실패 지점 → 마무리).

## 3. 오개념 카탈로그 (Misconception Catalog)

각 오개념은 **예상 → 탐지 → 교정** 3단으로 구조화한다. 태그 `[3축:인식/학습/심리]`는 accessibility-lens 기준 주 영향 축. 태그 `[실패지점:A~D]`는 concept-translator가 제공한 주방 은유 실패 지점 매핑(A=기억 휘발, B=순간 복제, C=위계 약화, D=도구 오남용).

### M1. "AI가 스스로 판단한다 (= AI에게 의지가 있다)"

- **예상:** 뉴스·SF 영향으로 초심자 다수가 기본값으로 가짐. Lv.1~Lv.2 초반에 반드시 만남.
- **탐지 질문:** "에이전트가 '원하는 결과'를 가진다고 생각하시나요?"
- **교정 스크립트 2단 분리 (R-LXD-05 반영):**
  - **(1) 개념 교정 문장 — 주):** *"AI는 '원한다'고 느끼지 않습니다. 설계된 역할 안에서 다음 단어·다음 행동을 확률적으로 고릅니다. 주방의 AI 셰프는 '이 요리를 만들고 싶다'가 아니라 '이 주문서에 따라 다음 동작을 선택한다'입니다. '판단'이라는 말을 쓰더라도, 그 판단은 **미리 설계된 선택지 중 고르는 일**입니다."*
  - **(2) 안심 문장 — 부):** *"그래서 AI가 돌발 행동으로 여러분을 놀라게 하는 일은 설계 범위를 벗어나기 어렵습니다."*
- **[3축:학습 주 · 심리 부]** — Rev.1에서 심리축 단독 태깅했으나 근본은 "의지·판단 개념의 오해"(학습축)임을 수용. 교정의 중심은 개념 설명, 안심은 부가.

### M2. "에이전트는 사람 직원이다 (= 24시간 출근해 있다)"

- **예상:** "AI 동료"·"AI 직원" 같은 친숙한 번역이 만들어내는 부작용. (**Rev.3 주의:** 주방 은유 기반 학습 경로에서는 "AI 동료" 용어 자체를 쓰지 않는다. "AI 셰프"로 통일.)
- **탐지 질문:** "지금도 에이전트가 어딘가에서 돌아가고 있을까요?"
- **교정 스크립트:** *"에이전트는 호출될 때만 깨어나는 존재입니다. 주방은 영업 시간에만 열리고, 영업이 끝나면 셰프는 사라집니다. 다음 영업에는 기억을 가진 채로 오는 게 아니라, 새로 열쇠를 꽂는 겁니다."*
- **[3축:학습][실패지점:A]**

### M3. "하네스는 AI 제품이다 (= 사고팔 수 있는 앱이다)"

- **예상:** "하네스만 도입하면 된다"는 관용 표현에 의해 강화됨. concept-translator 4.2에서 명시적 경고.
- **탐지 질문:** "하네스를 '사러' 어디로 가야 할까요?"
- **교정 스크립트:** *"하네스는 제품이 아니라 **사람이 짜고 매일 고쳐 쓰는 운영 매뉴얼**입니다. 주방 운영 매뉴얼 자체를 팔지 않듯이요. 주방을 어떻게 꾸리고 매일 어떻게 운영할지를 합의한 문서 뭉치에 가깝습니다."*
- **[3축:인식][실패지점:-]**

### M4. "오케스트레이터가 혼자 다 한다 (= 가장 똑똑한 AI 하나)"

- **예상:** "지휘자"라는 번역이 단독자 이미지를 강화함. Lv.2 L2-3에서 방어적으로 언급해야 함.
- **탐지 질문:** "오케스트레이터가 실제로 요리를 만들까요?"
- **교정 스크립트:** *"헤드셰프는 요리를 직접 만들지 않는 경우가 많습니다. 받은 주문을 '누가 무엇을 할지'로 나누고, 결과를 다시 접시에 모읍니다. 오케스트레이터도 같습니다. 조율자이지 만능 셰프가 아닙니다."*
- **[3축:학습][실패지점:C]**

### M5. "에이전트들은 위계가 있고 상명하복한다"

- **예상:** 주방·군대 은유에서 유래. concept-translator가 명시한 실패 지점 C.
- **탐지 질문:** "에이전트는 오케스트레이터에게 '아니오'라고 할 수 있을까요?"
- **교정 스크립트:** *"실제 주방은 위계적이지만 이 주방은 코멘트가 오가는 주방입니다. 에이전트들은 서로 피드백을 주고, 비판하고, 재작업을 요청합니다. 헤드셰프가 독재하는 그림이 아닙니다."*
- **[3축:학습][실패지점:C]**

### M6. "AI는 한 번 배우면 계속 기억한다"

- **예상:** 사람 직원을 떠올릴 때 자동으로 따라오는 가정. 메모리 설계 개념 도입 전 반드시 해체.
- **탐지 질문:** "오늘 AI가 배운 걸 내일도 기억할까요?"
- **교정 스크립트:** *"기본적으로는 잊습니다. 매 영업마다 셰프가 바뀐다고 생각하면 가깝습니다. 그래서 '조리 노트(메모리 파일)'에 중요한 것을 기록해두고, 다음 영업 때 먼저 읽게 만듭니다. 그 설계가 바로 메모리 설계입니다."*
- **[3축:학습][실패지점:A]**

### M7. "하네스는 사람 없이 돌아간다"

- **예상:** '자율·자동·에이전틱' 단어에서 유도되는 기대. 심리 장벽(두려움)도 동반.
- **탐지 질문:** "하네스가 도입되면 사람이 할 일이 사라지나요?"
- **교정 스크립트:** *"하네스는 '누가 무엇을 어떻게 할지'를 **사람이 짭니다**. 주방 운영 매뉴얼을 쓰고 매일 고치는 건 사람입니다. 영업 중에도 주방장·사장이 개입합니다. 하네스가 하는 일은 '반복 단계를 AI 팀이 맡게 하는 것'이지 '사람이 사라지는 것'이 아닙니다."*
- **[3축:심리][실패지점:-]**

### M8. "AI가 자기 복제로 무한대 확장되면 무섭다 / 반대로 AI도 한 번에 한 가지만 한다"

- **예상:** 양극단 오해가 공존. 어떤 이는 두려움, 어떤 이는 과소평가.
- **탐지 질문:** "에이전트는 한 번에 몇 명 일할 수 있을까요?"
- **교정 스크립트 2단 분리 (R-LXD-05 반영):**
  - **(1) 개념 교정 문장 — 주):** *"이 주방은 셰프를 '순간 복제'할 수 있는 마법 주방입니다. 그래서 동시에 수십 개 주문을 처리할 수 있지만, 복제할수록 전기·가스 미터기(토큰)가 빨리 돕니다. 무한도 아니고 한 명도 아닙니다. **설계된 범위 안에서** 동시 실행됩니다."*
  - **(2) 안심 문장 — 부):** *"복제 수는 사람이 미리 정해둔 상한 안에서만 가능합니다. 엉뚱한 방향으로 폭발적으로 늘어나지 않습니다."*
- **[3축:학습 주 · 심리 부][실패지점:B]** — Rev.1의 심리축 단독 태깅을 학습 주 + 심리 부로 정정. 핵심은 "동시성의 작동 원리"를 이해시키는 것, 안심은 결과로 따라온다.

### M9. "프롬프트는 검색어와 같다 (= 짧고 정확한 단어)"

- **예상:** 검색 경험에서 온 잔존 습관.
- **탐지 질문:** "AI에게 '여름 휴가 정리'라고만 쓰면 충분할까요?"
- **교정 스크립트:** *"프롬프트는 검색어가 아니라 **주문서**입니다. 누구에게·어떤 상황에서·어떤 형식으로·무엇을 피하고 싶은지까지 적어야 셰프가 원하는 접시를 냅니다. 짧게 쓸수록 잘 나오는 게 아닙니다."*
- **[3축:학습][실패지점:-]**

### M10. "컨텍스트 윈도우가 크면 무한히 기억한다"

- **예상:** '긴 컨텍스트' 마케팅 문구의 부작용. Lv.3에서 반드시 짚어야 함.
- **탐지 질문:** "컨텍스트 100만 토큰이면 기억 문제는 사라진 건가요?"
- **교정 스크립트:** *"조리대가 아무리 넓어도 영업이 끝나면 치웁니다. 컨텍스트가 커져도 '세션 안'에서만 유효하고, 세션 밖에선 여전히 기억이 사라집니다. 그리고 넓은 조리대일수록 주의 분산도 커집니다. 크다 = 좋다 가 아닙니다."*
- **[3축:학습][실패지점:A]**

### M11. "AI가 도구(Tool)를 자유롭게 쓴다 = 뭐든 시킬 수 있다"

- **예상:** 이메일·캘린더·파일 조작 데모를 보고 생기는 과신. concept-translator 실패 지점 D.
- **탐지 질문:** "AI가 제 이메일을 잘못 보낼 일은 없나요?"
- **교정 스크립트 2단 분리 (R-LXD-05 반영):**
  - **(1) 개념 교정 문장 — 주):** *"AI 셰프는 칼을 잘못 잡을 수 있습니다. 그래서 하네스 설계에는 '어떤 도구를 쓸 수 있는지(권한)', '어떤 행동은 사람 확인을 거쳐야 하는지(확인 지점)', '실패했을 때 어떻게 돌이킬지(복구 경로)' 세 가지가 반드시 들어갑니다. 도구를 줬다 = 전권을 준 것이 아닙니다."*
  - **(2) 안심 문장 — 부):** *"잘 설계된 하네스에서는 실수할 수 있는 행동 앞에 반드시 사람이 한 번 멈춥니다."*
- **[3축:학습 주 · 심리 부][실패지점:D]** — Rev.1의 심리축 단독 태깅을 학습 주 + 심리 부로 정정. "권한·확인·복구" 세 설계 요소를 아는 것이 핵심.

### M12. "하네스를 이해하려면 코딩을 알아야 한다"

- **예상:** 진입 전 심리 장벽의 1순위. Lv.1 초입에서 반드시 해체.
- **탐지 질문:** "프로그래밍 배운 적 없는데 이해할 수 있을까요?"
- **교정 스크립트:** *"하네스의 설계는 주로 **문장과 문서**로 이뤄집니다. '이 역할은 이런 일을 한다'를 자연어로 적는 일이 핵심입니다. 코딩은 도구를 붙일 때 일부 관여하지만, 개념 이해에는 필요하지 않습니다. 레시피를 읽는 것과 요리사 자격증을 따는 일은 다릅니다."*
- **[3축:심리][실패지점:-]**

### M13. "에이전트 = 챗봇 = AI 비서"

- **예상:** 이전 세대 용어의 잔존. concept-translator 4.1에서 명시 경고.
- **탐지 질문:** "챗봇과 에이전트의 차이를 말씀해 보시겠어요?"
- **교정 스크립트:** *"챗봇은 대화 상대입니다. 에이전트는 대화뿐 아니라 **도구를 써서 외부 세계에 행동**합니다. 메일을 보내고, 파일을 수정하고, 검색을 하고, 다른 에이전트에게 일을 넘깁니다. 주문을 받는 점원 vs 실제로 요리하는 셰프의 차이에 가깝습니다."*
- **[3축:인식][실패지점:-]**

### M14. "하네스는 대기업·전문가용이다. 나와 상관없다"

- **예상:** 심리 장벽. 특히 중장년·비기술직에서 강함. Lv.1 도입 훅에서 방어 필요.
- **탐지 질문:** "이게 왜 나에게 중요할까요?"
- **교정 스크립트 (R-LXD-12 반영):** *"AI에게 '그냥 알아서 해'가 아니라 '이 역할로 도와줘'라고 부탁하는 순간, 여러분은 이미 **하네스적인 태도**를 쓰고 있는 겁니다. 그 태도와, 실제로 여러 AI를 엮어 돌리는 **하네스 설계**는 다른 일입니다만, 그 태도를 갖는 것만으로도 일상에서 AI를 더 잘 쓸 수 있습니다. '이해'는 직업·규모와 무관하게 이득입니다."*
- **[3축:심리][실패지점:-]**
- **Rev.2 주의:** Rev.1의 "작은 하네스를 만든 셈"은 Lv.3 완료 기준 #4(하네스와 챗봇/AI 비서 구분)와 내적 모순을 일으켰다(R-LXD-12). "하네스적 태도"로 수위 조정하여 **태도(attitude)**와 **설계물(artifact)**을 분리. 이 분리로 Lv.3의 경계 구분 연습이 흔들리지 않는다.

### M15. "하네스를 이해한다 = 만들 줄 안다"

- **예상:** 학습자가 Lv.3 말미에 과도한 자신감이나 과소 자신감 양쪽으로 빠질 수 있음. 기대 관리 필요.
- **탐지 질문:** "이 과정을 끝내면 하네스를 만들 수 있나요?"
- **교정 스크립트 (R-LXD-10 반영):** *"**개념 이해**와 **실제 설계**는 다른 능력입니다. 이 학습의 목표는 '타인에게 설명할 수 있다'까지입니다. 이해만 해도 하네스가 적용된 세상을 읽을 수 있고, 그것만으로도 충분합니다. **다만 여기서 더 나아가 실제로 하네스를 만들어보고 싶다면 별도의 실행 학습 경로가 있습니다.** 이해는 도착점이 아니라 선택 가능한 출발점입니다."*
- **[3축:심리][실패지점:-]**
- **Rev.2 주의 (R-LXD-10 반영):** Rev.1의 "이해만으로 충분"은 자기결정이론상 유능감을 느낀 학습자에게 "그만하라"는 메시지로 읽혀 성장 동기를 저해할 수 있다는 지적 수용. 개방 문구를 추가해 "여기까지도 충분하다"와 "더 나아갈 길이 있다"를 **양립 가능한 선택지**로 제시.

### M16. "하네스는 한 번 짜면 끝난다 (= 완성되면 고칠 일 없는 문서)"  **[Rev.3 신설 · R-CT-02 관련]**

- **예상:** "설계도"라는 단어에 묶여있던 초기 인상의 잔존. 특히 엔지니어링 배경 청중은 "아키텍처 문서 = 한 번 쓰면 끝"이라는 습관을 투사한다.
- **탐지 질문:** "하네스를 한번 잘 만들어 두면 이후엔 그냥 돌아가나요?"
- **교정 스크립트 2단:**
  - **(1) 개념 교정:** *"하네스는 '설계도'보다 '**운영 매뉴얼**'에 가깝습니다. 식당 매뉴얼이 계절 메뉴·새 셰프·새 손님층을 만나 매일 조금씩 고쳐지듯, 하네스도 **사용하면서 계속 개정됩니다**. 한 번 짜서 끝나는 문서가 아니라, 살아 있는 문서입니다."*
  - **(2) 안심 문장:** *"고치기 쉽게 짜두는 것 자체가 좋은 하네스의 조건입니다. 처음부터 완벽할 필요 없습니다."*
- **[3축:학습 주 · 심리 부][실패지점:-]** — concept-translator v2의 매뉴얼 재정의에 따른 신규 오개념. Lv.3의 L3-8에서 정면 교정.

### M17. "MCP는 배송 시스템·파이프라인 같은 것"  **[Rev.3 신설 · R-CT-03 관련]**

- **예상:** concept-translator v1의 "배송 파이프라인" 은유에 노출된 청중. 외부 자료·뉴스에서도 여전히 등장할 수 있어 잔존 오개념으로 예상.
- **탐지 질문:** "MCP가 뭐냐"는 질문에 "배송", "파이프라인", "전송" 같은 단어를 먼저 꺼내는가?
- **교정 스크립트:** *"MCP는 배송 시스템이 아니라 **USB-C 같은 표준 포트**입니다. 무엇을 전달하는 기계가 아니라, 서로 다른 도구가 **같은 규격으로 꽂히도록** 정한 규칙입니다. 덕분에 새 장비를 주방에 들일 때마다 배선을 다시 짤 필요가 없습니다."*
- **[3축:학습][실패지점:-]** — concept-translator v2의 은유 교체에 따른 과도기 오개념. Lv.2 L2-5에서 정면 다룸.

### M18. "헤드셰프가 명령하면 셰프는 따른다 (= 실제 주방 위계를 그대로 투사)"  **[Rev.3 신설 · R-CT-01 관련]**

- **예상:** 주방 은유 자체가 지닌 문화적 위계 이미지(드라마 '하얼빈 주방장' 식 묘사)에서 자동으로 따라오는 오해. M5("에이전트는 상명하복")보다 특정적: **헤드셰프의 권위**에 초점.
- **탐지 질문:** "헤드셰프 AI가 '이거 해'라고 하면 다른 AI는 무조건 해야 하나요?"
- **교정 스크립트 2단:**
  - **(1) 개념 교정:** *"이 주방은 **코멘트가 오가는 주방**입니다. 헤드셰프(오케스트레이터)가 방향을 제시하지만, 다른 셰프(에이전트)들은 '이건 이렇게 하는 게 낫지 않나요?'라고 되묻고, 심지어 헤드셰프의 판단에 '다시 해달라'고 요청할 수 있습니다. 권위로 굴러가는 주방이 아니라 **피드백으로 굴러가는 주방**입니다."*
  - **(2) 안심 문장:** *"그래서 AI 팀의 결과가 한 AI의 독단으로 치우치지 않습니다. 서로가 서로를 점검합니다."*
- **[3축:학습][실패지점:C]** — M5와 구분: M5는 "일반적 상명하복 오해", M18은 "헤드셰프 = 절대 권위 오해". 두 오개념은 실패지점 C를 서로 다른 각도에서 공격한다. Lv.2 L2-3(오케스트레이터)과 Lv.3 L3-6(은유 실패지점)에서 이중 방어.

| 실패지점 B (순간 복제) | M8 | ✅ |
| 실패지점 C (위계 약화) | M4, M5, **M18** | ✅ (Rev.3: M18 신설로 C축이 3개 독립 오개념 커버 — M4=헤드셰프 독재, M5=에이전트 상명하복 일반, M18=헤드셰프 권위 특정) |
| 실패지점 D (도구 오남용) | M11 | ✅ |

**Rev.3 총계:** 18개 오개념. 학습축 주 12건 / 심리축 주 4건 / 인식축 2건 + 심리 부 이중 태깅 4건. 실패지점 C는 v1의 중복 우려를 넘어 3개의 서로 다른 각도에서 독립 교정 가능해짐.

**Rev.2 변경:** Rev.1은 심리축 7건/학습축 6건이었으나, critical-reviewer R-LXD-05 지적 수용하여 M1·M8·M11을 "학습 주 + 심리 부" 이중 태깅으로 재분류. 이제 학습축 9건·심리 주 4건·심리 부 3건. 심리축이 과잉 팽창해 교정이 "안심시키기"로 기울던 문제 해소. 교정 스크립트도 2단(개념 교정 + 안심)으로 분리 명시.

**실패지점 C 중복 인지:** M4(오케스트레이터 독재)와 M5(에이전트 상명하복)가 동일 실패지점 C에 매핑된다는 외부 지적(Minor) 수용. 두 오개념은 "위계" 개념을 각기 다른 방향(헤드셰프→셰프, 셰프→셰프)에서 오해한다는 점에서 독립 교정 가치가 있으나, **실패지점 4종이 오개념과 1:1로 매핑되지는 않음**을 명시하여 synthesis-lead가 제안서에 잘못된 인상을 주지 않도록 한다.

## 4. 학습 자료 포맷 매트릭스

각 개념에 대해 "어떤 매체가 가장 이해를 잘 유도하는가"를 지정. 제품 인터페이스가 아닌 **학습 자료 매체** 관점.

> ⚠️ **경험상 가설 — 검증 전 참고 (R-LXD-07 반영).** 본 매트릭스의 매체 선정은 교육 설계 경험에 기반한 **가설**이며, 실제 청중 테스트·A/B 측정으로 검증되지 않았다. 제작 의사결정에 사용할 때는 "첫 번째 후보"로만 취급하고 소규모 파일럿으로 확인 후 확장하기를 권장한다. 특히 아래 두 가정은 이견이 있을 수 있다:
> - **카드뉴스를 "중난이도·고도달력"으로 본 가정** — 일반 원칙상 짧은 매체일수록 압축 난이도가 높아질 수 있다. Rev.1에서 단정한 "저렴한 매체로 낮은 레벨을 덮는다"는 전제는 매체 제작 품질을 전제로 할 때만 성립한다.
> - **영상 매체의 우월 가정** — 심리축 낮추기에 영상이 유리하다는 가정은 청중의 기기 환경·연령대에 따라 달라진다.

| 개념 | 최적 매체 (1순위) | 보조 매체 (2순위) | 비권장 매체 | 이유 |
|---|---|---|---|---|
| P1. AI는 도구 | 3컷 카드뉴스 | 1분 영상 | 긴 글 | 이미 알고 있음. 환기만 필요 |
| P2. 프롬프트 감각 | 짧은 Q&A 대화체 | 비교 예시 글 | 영상 | "이렇게 쓰면 vs 저렇게 쓰면" 대비가 핵심 |
| P3. 팀 분업 직관 | 비유 스토리 (영상/만화) | 카드뉴스 | 다이어그램 | 감정·공감이 학습 고정에 유효 |
| P4. AI의 한계 | 인포그래픽 (용량 막대) | 짧은 글 | 영상 | 수치·비교 시각화가 강함 |
| C1. 에이전트 | 1분 영상 (주방 장면) | 인포그래픽 | 추상 다이어그램 | 구체 장면이 추상 개념 고정에 가장 유효 |
| C2. 스킬 | 카드뉴스 (레시피 카드 시각화) | Q&A | 긴 글 | "카드"라는 매체 자체가 은유 |
| C3. 오케스트레이터 | 3분 영상 (흐름 애니메이션) | 다이어그램 | 카드뉴스 | 흐름·분기가 핵심이라 애니메이션이 적합 |
| C4. 하네스 | 10분 발표/글 + 인포그래픽 (청사진) | 3분 영상 | 1분 쇼츠 | 본 개념은 깊이 필요 |
| E1. 서브에이전트 | 비유 스토리 (**게스트 셰프** 에피소드) | 영상 | 다이어그램 | 에피소드형이 이해에 유리 |
| E2. 컨텍스트 윈도우 | 인포그래픽 (조리대 시각화) | 영상 | 긴 글 | 공간적 메타포라 시각이 유리 |
| E3. 메모리 | 비유 스토리 ("내일의 셰프에게 쓰는 노트") | 긴 글 | 다이어그램 | 서사형이 가장 오래 남음 |
| E4. 도구·MCP | 인포그래픽 (**USB-C 표준 포트** 시각화) | Q&A | 영상 | 구조적 개념은 도해가 유리. Rev.3: "어댑터·콘센트"에서 "USB-C"로 통일 |
| E5. 하네스 팀 | 단체 사진식 비유 이미지 + 글 | 영상 | 카드뉴스 | 팀 구성을 한눈에 |
| X. 은유의 실패 지점 4종 | 표 + 긴 글 | 강연 슬라이드 | 영상 | 비교·대조가 많아 정적 매체가 유리 |

### 매체별 제작 난이도 vs 도달력 매트릭스 (참고)

```
도달력 ↑
  │
  │  3분 영상 ───────── 카드뉴스
  │  ●(고난이도)         ●(중난이도)
  │
  │  10분 발표         1분 영상
  │  ●(고난이도)         ●(중난이도)
  │
  │  인포그래픽        Q&A 글
  │  ●(중난이도)         ●(저난이도)
  │
  └──────────────────────────────── 제작난이도 →
```

권장 우선순위: **카드뉴스 → 1분 영상 → 인포그래픽 → 3분 영상 → 긴 글 → 10분 발표**. 저렴한 매체로 낮은 레벨을 덮고, 핵심 심화는 비싼 매체로.

## 5. 단계별 자가 점검 질문 (Comprehension Check)

각 레벨 종료 시 학습자가 스스로 풀어보는 질문. 정답 예시를 함께 둔다.

### Lv.1 자가 점검 (5문항)

> 각 문항은 30초 이내에 답할 수 있어야 통과.

1. **주관식:** "하네스가 뭐야?"라고 친구가 물었을 때 한 문장으로 답하시오.
   - *정답 예:* "사람이 짜서 여러 AI가 역할을 나눠 함께 일하게 만드는 **운영 매뉴얼**이야. 매일 조금씩 고치면서 쓰는 거야." *(Rev.3: 주어 "사람이" 포함 + "매뉴얼" 용어 사용이 통과 표시)*
2. **O/X:** 하네스는 앱스토어에서 살 수 있는 AI 제품이다. ( )
   - *정답:* X — 하네스는 설계 방식·문서이다.
3. **선택:** 에이전트에 가장 가까운 것은?
   ① 챗봇 ② 역할이 부여된 AI ③ AI 자체 ④ 자동화 매크로
   - *정답:* ②
4. **주관식:** 한 주방에 요리사 한 명만 있을 때 생기는 문제를 한 문장으로 적으시오.
   - *정답 예:* "복잡한 코스를 혼자 다 하면 헤매거나 놓친다."
5. **체크:** 아래 중 하네스의 본질에 가까운 설명은?
   - [ ] AI를 더 똑똑하게 만드는 기술
   - [x] 여러 AI가 엇갈리지 않게 일하도록 엮는 설계 기술
   - [ ] AI가 스스로 회사를 운영하게 하는 기술

### Lv.2 자가 점검 (7문항)

1. **주관식:** 하네스의 다섯 요소를 나열하고 각각 한 문장으로 설명하시오. (에이전트/스킬/오케스트레이터/도구·MCP/메모리)
2. **순서 배열:** 아래를 순서대로 배열하시오.
   `결과 통합` / `에이전트 실행` / `오케스트레이터가 분배` / `사용자에게 전달` / `프롬프트 입력`
   - *정답:* 프롬프트 입력 → 오케스트레이터가 분배 → 에이전트 실행 → 결과 통합 → 사용자에게 전달
3. **교정 과제:** 다음 문장의 잘못된 부분을 고치시오.
   *"AI 에이전트는 출근해서 하루 종일 일하다가 퇴근한다."*
   - *정답 예:* 에이전트는 호출될 때만 깨어나고, 작업이 끝나면 사라진다. 기억도 함께 사라진다.
4. **주관식:** 내가 오늘 AI에게 시킨 작업이 "하네스적"이었는지 판별하고 이유를 쓰시오.
5. **서술형 (R-LXD-09 반영):** 오케스트레이터가 수행하는 역할 **3가지 이상**을 자기 말로 서술하시오. (힌트 금지)
   - *정답 예시:* 요청 해석 / 작업 분배 / 에이전트 선택 / 결과 통합 / 다음 단계 판단 중 3개 이상.
   - *미통과 예시:* "모든 요리를 다 한다" / "AI를 감독한다"(구체성 부족) / "혼자 한다"
   - **변경 이유:** Rev.1의 4지선다는 변별도가 낮았다("모든 요리 직접 수행"이라는 명백한 오답 보기가 하나뿐). 서술형은 M4(오케스트레이터 독재) 오개념을 능동적으로 드러내 탐지한다.
6. **O/X:** 컨텍스트 윈도우가 크면 AI는 영구히 기억한다. ( )
   - *정답:* X
7. **주관식:** MCP를 한 문장으로 설명하시오. (주방 은유 허용)
   - *정답 예 (Rev.3):* "AI 주방이 외부 장비·서비스와 연결될 때 쓰는 **USB-C 같은 표준 포트**."
   - *미통과 예시:* "배송 시스템" / "파이프라인" — 이는 M17 오개념에 해당.

### Lv.3 자가 점검 (10문항)

1. **실연:** 10분 분량으로 하네스를 타인에게 설명하는 시나리오를 구조화하시오 (도입 훅 → 다섯 요소 → 실패 지점 → 마무리).
2. **주관식:** 주방 은유의 4대 실패 지점을 각각 쓰시오.
   - *정답:* 기억 휘발 / 순간 복제 / 위계 약화 / 도구 오남용
3. **구분 연습:** 다음 중 하네스가 **아닌** 것을 골라 이유를 쓰시오.
   - (a) 다섯 에이전트가 회의록을 정리·요약·결정사항 카드화 하는 구성
   - (b) "ChatGPT에게 요약해줘"라고 한 번 부탁하는 것
   - (c) 이메일 자동 분류 매크로
   - *정답:* (b), (c) — (b)는 단일 에이전트 호출이라 하네스 구조 없음, (c)는 규칙 기반 자동화.
4. **교정 (R-LXD-06 반영 — 짝짓기 포맷):** 아래 15개 오개념 목록(M1~M15)을 보며 **최소 1개 이상**을 골라 각각에 대해 교정 문장을 자기 말로 작성하시오. 2~3개를 하면 더 좋음. "무작위 시험" 방식은 사용하지 않음.
   - **변경 이유:** Rev.1의 "무작위 3개 시험" 포맷은 M15("이해만 해도 충분")와 내적 충돌을 일으켜 심리 부담을 재발시켰다(R-LXD-06). 목록을 보며 스스로 고르는 방식은 심리 부담을 낮추면서도 교정 능력을 드러낸다.
5. **주관식:** 메모리 설계가 왜 하네스에서 중요한가? (셰프 기억 휘발 비유 활용)
6. **비교:** 챗봇 / AI 비서 / 에이전트 / 자동화 매크로 네 가지를 구분하여 표로 작성.
7. **주관식:** 서브에이전트와 에이전트의 차이를 **게스트 셰프** 은유로 설명하시오. (Rev.3: "외주 요리사" → "게스트 셰프" 용어 통일)
8. **적용:** 주변의 반복 업무 한 가지를 골라 "어떤 에이전트 · 어떤 스킬 · 어떤 도구가 필요할지" 자연어로 스케치 (구현 아님, 기술 언어 금지, 일상 언어 사용).
9. **경계:** 하네스가 사람을 대체하지 않는다는 것을 어떻게 설명할 것인가?
10. **메타:** 이 학습의 목표는 "만들 수 있다"가 아니라 "설명할 수 있다"이다. 왜인가?

### 6.2 진행자 핵심 유의점 5가지

1. **은유를 섞지 말 것.** 한 세션 안에서 주방·영화·스포츠를 혼용하면 청자 머릿속에서 이미지가 깨진다. 한 은유로 끝까지 간다.
2. **새 용어를 한 번에 3개 초과 도입하지 말 것.** 넘으면 암기 부담으로 전환되어 이해가 멈춘다.
3. **"모르겠다"를 환영할 것.** 질문 없는 청중이 이해한 게 아니라, 포기한 것일 수 있다. "지금 어디서 끊겼는지"를 물어라.
4. **"AI가 생각한다 / 판단한다" 같은 의인화 표현을 교정 없이 쓰지 말 것.** 매 발화마다 "응답을 생성한다", "설계된 범위 안에서 선택한다"로 교체.
5. **심리 장벽을 과소평가하지 말 것.** "이해만으로 충분합니다"를 최소 3번 반복. 만들 줄 알아야 한다는 압박을 세션 중에 주지 않는다.

## 7. concept-translator 산출물과의 일관성 검증

본 학습 경로는 concept-translator 산출물을 아래와 같이 활용·참조했다:

- **중심 은유 "주방 브리게이드"** — Lv.1~Lv.3 전 과정에서 1순위 은유로 채택. 혼합 금지 원칙 준수.
- **L1/L2/L3 레이어 9개 개념** — 개념 지도의 C1~C4, E1~E5에 1:1 매핑되어 레슨 분량 산정 기준이 됨.
- **실패 지점 4종 (A~D)** — 오개념 카탈로그의 M2, M6, M10(A) / M8(B) / M4, M5(C) / M11(D)에 매핑.
- **피해야 할 오역 8종 (4.2 섹션)** — 교육자 가이드의 "Don't List"와 오개념 카탈로그의 M3, M7, M9, M13 등에 반영.
- **1분/3분/10분 스크립트** — 각 레벨의 통합 전달물 템플릿으로 활용.

### 은유 공백 점검

현재 식별된 공백 **없음**. 주방 은유 + 보조 3종(영화/스포츠/스튜디오)으로 페르소나 3인 모두 커버 가능. 단, **E3(메모리)**를 주방 은유로 설명할 때 "영업 끝나고 내일의 셰프에게 노트를 쓴다"는 장면이 **은유 D(기억 휘발)**의 교정을 겸하는 이중 사용인 점은 강력하다. concept-translator에게 추가 요청 없음.

향후 필요가 생길 지점: **"하네스는 어떻게 진화·업데이트되는가"**에 대한 은유 — ~~주방 설계도가 계절마다 바뀐다~~ **(Rev.3 해결)** concept-translator v2에서 하네스 정의 자체가 "매일 고쳐 쓰는 운영 매뉴얼"로 재정의되었고, Lv.3의 L3-8 레슨이 이 속성을 다루도록 신설되었다. 은유 공백 없음.

## 8. Accessibility Score (3축 자체 채점 · Rev.2)

accessibility-lens 지침에 따라 본 산출물을 3축 5점 척도로 자체 채점한다. Rev.1의 자체 4.33/5는 외부 검증 없는 낙관으로 critical-reviewer가 2.8~3.3(평균 3.0)으로 재평가했다. Rev.2에서는 **그 재평가를 baseline으로 수용**하고, 본 개정이 적용된 후의 목표 점수를 아래와 같이 제시한다.

Accessibility Score — Rev.2 (목표 · 개정 반영 후)

Rev.1 외부 재평가   → 3.00 / 5 (critical-reviewer 추정)
Rev.2 자체 목표     → 3.50 / 5 (현장 검증 전 잠정)

- 인식 낮춤(Awareness): ⬤⬤⬤○○ (3/5 · Rev.1 추정 4→3 하향)
  Lv.1 1분 경로가 존재를 알리지만, "하네스"라는 단어 자체의 첫 노출을
  외부 채널에 의존하는 구조적 한계는 본 학습 자료로 해결 불가.
  홍보·채널 전략은 synthesis-lead의 별도 과제.
