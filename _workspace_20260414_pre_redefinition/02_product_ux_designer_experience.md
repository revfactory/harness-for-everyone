## 0. 설계 대전제 — 왜 "터미널이 아닌 주방"인가

"설치하고 명령어 치면 됩니다"는 30~50대 비개발자에게 **심리적 셔터**를 내린다. 우리는 그들이 **이미 쓰고 있는 4개의 문(門)**으로 주방을 드나들게 한다.

- **모바일 앱** — 주방을 **눈으로 보는** 정문
- **카카오톡** — 주방에 **주문서를 던져 넣는** 쪽문
- **음성(스피커/전화)** — 손이 젖어 있을 때 **소리쳐 주문하는** 뒷마당 창
- **브라우저 익스텐션** — 이미 **일하던 책상 옆에 붙은** 주방 창구

한 사람이 4개 문을 동시에 쓰는 게 아니다. **페르소나별로 주 출입문이 다르고**, 나머지는 보조로 배치된다.

## 1. 4채널 경험 아키텍처

### 1-1. 채널별 역할 분담 매트릭스

| 채널 | 주방 은유상 위치 | 주 사용자 | 핵심 강점 | 구조적 약점 | 담당 시나리오 |
|------|----------------|----------|---------|-----------|-------------|
| **모바일 앱 (iOS/Android)** | 주방 **정문 (홀 전체 조망)** | 30~40대 자영업자·프리랜서 | 실시간 조리 시각화, 권한 관리, 메모리(포스트잇) 편집 | 설치 저항, 앱 전환 부담 | 첫 가입, 조수 부르기, 권한 설정, 결과 승인, 결제 |
| **카카오톡 채널** | 주방 **쪽문 (주문 접수창)** | 30~50대 전반. 특히 앱 설치 기피층 | 설치 0, 문턱 최저, 한국 대중이 **이미 매일 쓰는 손가락 습관** | 긴 카드 UI·파일 첨부·실시간 진행바 한계, 메시지 50초 응답 제약 | 매일의 반복 주문, 알림, 짧은 결과 확인, **try-before-install** |
| **음성 (AI 스피커/전화 ARS)** | 주방 **뒷마당 창 (핸즈프리)** | 50~60대, 운전 중, 손 못 쓰는 노동 환경(조리·돌봄·운전) | 글자 몰라도 됨, 이동 중 사용, **문해력 장벽 제거** | 컨텍스트 부재, 복잡한 선택 어려움, 프라이버시 공포 | 기록 받아쓰기, 단일 명령, 알림 청취, 확인 질문 |
| **브라우저 익스텐션 (크롬/엣지)** | 주방 **창구 (책상 옆 패스스루)** | 30~40대 지식 노동자·PC 기반 실무자 | 현재 보고 있는 **웹 페이지를 재료로** 바로 전달, 클립보드/탭 자동 포착 | 모바일 불가, 로그인 상태 의존 | 리서치 요약, 메일 초안, 문서 변환, 표 정리 |

### 1-2. 채널 간 연결 원칙 — "같은 주방, 다른 문"

- **단일 계정·단일 메모리·단일 권한**: 어느 문으로 들어오든 **같은 포스트잇(메모리)**과 **같은 열쇠 꾸러미(권한)**를 본다. 채널을 바꿨다고 조수가 낯설어지지 않는다.
- **채널 승급(Escalation) 규칙**:
  - 카톡에서 긴 작업이 시작되면 → "결과가 길어져요. 앱에서 열어볼까요?" 링크 카드 제시
  - 음성에서 금전·외부 발송이 임박하면 → "앱에서 마지막 확인해 주세요" 강제 승급
  - 브라우저에서 민감 권한 요청 → 모바일 앱 푸시 2차 승인
- **말투 공통 규약**: 모든 채널에서 조수의 1인칭을 "**저**", 사용자 호칭은 "**사장님**"(자영업 톤) 또는 사용자 설정값으로 통일. 사투리/존대 수준은 온보딩에서 택일.

### 1-3. 채널 조합 프리셋(가입 시 질문 1개로 결정)

**Q. "주로 어디서 주문을 넣으실래요?"**

| 프리셋 | 조합 | 대상 |
|-------|------|------|
| 🔔 **카톡만** | 카톡 채널 (앱 설치 생략) | 앱 스토어 공포층, 어르신 |
| 📱 **앱 + 카톡** | 앱(주), 카톡(알림·간단주문) | 자영업자, 프리랜서 기본값 |
| 💻 **앱 + 브라우저** | 앱(모바일), 확장(PC 업무) | 지식 노동자 |
| 🎙️ **음성 + 카톡** | 스피커 또는 전화번호, 카톡(결과 확인) | 돌봄·운전·조리 중 핸즈프리 |

## 2. 첫 30분 여정 — Aha Moment 설계

**목표 순간:** "내가 뭘 한 거지?"가 아닌 **"아, 헤드셰프한테 오늘 한 접대 조수 한 명 빌린 거구나"** 라는 자각.

### 2-1. 감정 곡선(30분 구간)

```
감정↑
 기대  [██]
 안심       [████]
 재미            [██████]
 놀람                 [████████]  ← Aha!
 소유감                    [██████████]
 └─0분──5분──10분──15분──20분──25분──30분→ 시간
   가입  말투  첫주문  조수합류  결과  이름지어주기
```

### 2-2. 단계별 화면 · 마이크로 카피 · 사용자 감정

| 시점 | 화면/이벤트 | 마이크로 카피 | 의도된 감정 |
|------|------------|--------------|------------|
| 0:00 | 스토어→앱 첫 화면 (또는 카톡 채널 추가) | "🍳 오늘의 주방에 오신 걸 환영합니다. **주방장(저)이 조수 한 명을 데리고 기다리고 있었어요.**" | 기대 |
| 0:30 | 호칭·말투 선택 (4택) | "저를 뭐라고 부를까요? ① 사장님(가게) ② OO님(일감으로) ③ 선생님(돌봄) ④ 직접 입력 / 저는 어떻게 말할까요? ① 친근하게 ② 정중하게" *(T-5 반영)* | 주도권 |
| 1:30 | **튜토리얼 주방 한 컷** (concept-translator 구도 A) | "여기는 주방이에요. 왼쪽이 저(헤드셰프), 뒤에 있는 앞치마들이 조수예요. 앞쪽 카운터에 **주문서 한 장 놓아보실래요?**" | 공간감 |
| 3:00 | **샌드박스 첫 주문** (설치 없이도 실행 가능) | 예시 버튼 3개: ①"지난주 받은 메일 중요한 것만 추려줘" ②"이 유튜브 영상 3줄로 요약" ③"내가 쓴 이 글 맞춤법 봐줘" | 안심 (내 데이터 아직 안 줌) |
| 3:30 | **조수 입장 애니메이션** | "조수 **'정리조수'**가 앞치마를 매고 들어왔어요. 주문서를 읽고 있어요…" (실시간 조리 진행바) | 재미 |
| 5:00 | **조리 진행 카드** | "🔪 재료 다듬는 중 (30%)" → "🔥 조리 중 (70%)" → "🍽️ 플레이팅 (95%)" | 시각적 만족 |
| 6:30 | **첫 결과 접시** | "여기 나왔습니다. 🍽️ 맛 좀 봐주세요. (위/아래 스와이프로 평가)" | 놀람 |
| 8:00 | **"조수에게 이름 지어주기" (Aha 트리거)** | "방금 그 조수, 앞으로도 자주 부르실 것 같아요. **이름을 지어주실래요?** (예: 지원이, 민수, 정리요정…) 이름은 사장님 편하시라고 붙이는 거예요. **기억은 주방 벽면 포스트잇**에 남아서 어느 조수든 꺼내 읽을 수 있어요." *(T-4 반영)* | 소유감 / Aha |
| 10:00 | 메모리 화이트보드 첫 기록 | "사장님이 **'존댓말 써'**라고 하셨네요. 벽면 포스트잇에 붙여둘게요. 다음에도 기억할 거예요." | 통제감 |
| 15:00 | **권한(열쇠) 첫 요청** | "이번 일은 **🔑 초록 열쇠(메일 읽기)**가 필요해요. 빌려주실래요? / 언제든 다시 뺏을 수 있어요." | 신뢰 |
| 20:00 | 두 번째 조수 등장 (팀 감각) | "'정리조수' 혼자 벅찰 것 같아요. **'검색조수' 한 명 더 불러도 될까요?** (이번 접대만)" | 팀의 감각 |
| 25:00 | **저장된 단골 주문 만들기** | "이 주문, 매주 반복될 것 같아요. **'월요일 아침 주문'**으로 저장할까요? 매주 알아서 준비해 둘게요." | 자동화 체감 |
| 29:00 | **마무리 브리핑** | "오늘 주방 첫날이었어요. **조수 2명·포스트잇 3장·열쇠 1개**가 생겼어요. 내일부터는 이 팀이 기억하고 있을게요." | 완성감 |

### 2-3. Aha Moment 3중 설계

사용자가 "와 이거 된다"를 느끼는 지점은 **3단 로켓**으로 분리한다.

1. **1차 Aha (6분차) — 결과 품질**: 첫 주문의 결과가 "챗GPT보다 낫네"로 체감되는 순간. (팁: 샌드박스 예시는 챗GPT 단독보다 확실히 나은 **다단계 작업**으로 고른다 — 여러 소스 합치기, 단계별 검증 포함)
2. **2차 Aha (8분차) — 소유감**: 조수에게 이름을 붙이는 순간. "이건 내 팀이다"라는 감각.
3. **3차 Aha (25분차) — 시간 복제**: 단골 주문 저장. "앞으로 이걸 내가 안 해도 된다"의 체감.

### 2-4. 실패 시 회복 경로

- 5분차 이탈(튜토리얼 지침) → 카톡 푸시 "주방 아직 안 열어보셨어요. **30초 샘플**만 볼까요?" (링크 → 결과만 미리 보기)
- 샌드박스 결과 실패 → "**조수가 얼버무렸네요**. 다른 조수로 바꿔볼까요?" (환각을 '얼버무림'으로 번역 — 부록 A 규칙)
- 권한 요청 거부 → "괜찮아요. **열쇠 없이 할 수 있는 일**만 먼저 해볼게요." (기능 제한 모드로 전환, 재요청 금지)

## 3. 핵심 화면 7개 와이어프레임

모든 와이어프레임은 모바일 앱 기준(세로 375×812 가정). 카톡/음성/브라우저 버전은 각 화면 말미에 "채널 변주" 표기.

### 화면 1 — 🏠 **주방 대시보드 (홈)**

```
┌─────────────────────────────────────┐
│ 🍳 오늘의 주방        [🔔3] [⚙️]    │  ← 상단바
├─────────────────────────────────────┤
│                                     │
│   좋은 아침이에요, 사장님.            │  ← 인사 카드 (시간대별)
│   오늘 주방에 불 들어왔어요. 🔥       │
│                                     │
├─────────────────────────────────────┤
│  이번 접대 요리팀 (3명)              │  ← 근무 중인 에이전트 (v2 세션 단위)
│  ┌───┐ ┌───┐ ┌───┐  [+ 한 명 더]   │
│  │지원│ │민수│ │경리│               │
│  │정리│ │검색│ │장부│               │
│  └───┘ └───┘ └───┘                 │
├─────────────────────────────────────┤
│  📋 오늘의 주문서                    │
│  ┌─────────────────────────────┐   │
│  │ [＋ 새 주문서 놓기]          │   │  ← 주 CTA
│  └─────────────────────────────┘   │
│                                     │
│  📌 자주 하는 주문                   │
│   • 월요일 아침 보고            ▷   │
│   • 거래처 메일 정리            ▷   │
│   • 수첩 사진 장부 정리         ▷   │
├─────────────────────────────────────┤
│  🔔 어제 주방에서                    │
│   · 김 사장님 단골노트 업뎃          │
│   · 세금계산서 3건 발행 완료         │
├─────────────────────────────────────┤
│ [🏠홈] [📜주문] [📒포스트잇] [🔑열쇠]│  ← 하단탭
└─────────────────────────────────────┘
```

**주방 은유 시각화:**
- 배경: 따뜻한 크림색(주방 벽 톤), 상단 오른쪽에 "🔥 주방 불 들어옴" 인디케이터 = 서비스 정상 표시
- 조수 카드: 각자 다른 앞치마 색. 눌러서 상세/교체/퇴근 보내기
- 하단탭 4개가 각각 (주방 정문 / 오늘의 주문서 다발 / 벽면 포스트잇 / 열쇠 꾸러미)

**마이크로 카피 예시:**
- "주방에 불 들어왔어요" (시스템 정상)
- "조수 한 명 더 불러올까요?" (에이전트 추가)
- "새 주문서 놓기" (신규 작업)
- 로딩 시: "잠시만요, 조수들이 앞치마 매는 중…"

**채널 변주:**
- 카톡: 대시보드 대신 **채팅방 상단 고정 카드**로 "이번 접대 요리팀 3명 · [새 주문]" 1줄
- 음성: "오늘의 주방입니다. 주문하실래요, 어제 결과 듣기 하실래요?"
- 브라우저: 팝업 아이콘. 현재 탭 썸네일이 "주문서 미리보기"로 떠 있음

### 5-2. 실수 복구(Undo) 설계 3중 안전망

1. **1차 — 실행 전 확인 (Prompt 단계)**: 빨강 열쇠가 걸린 주문은 주방이 접수 시 "**이 주문은 돈이 움직여요. 진짜 보낼까요?**" 재확인
2. **2차 — 실행 중 취소 (Run 단계)**: 진행 카드 상단에 상시 [⏸ 잠깐 멈춰]. 음성에서는 "스톱"/"잠깐"/"멈춰" 3개 모두 인식
3. **3차 — 실행 후 되돌리기 (Post 단계)**: 완료 후 60초 **되돌리기 창(Undo Window)** + 이후엔 "기록 보기"에서 취소 신청 가능. 메일·결제 등 외부 세계로 나간 액션은 **되돌릴 수 없음**을 실행 전에 명시

### 5-3. 투명성 — 조수가 지금 뭘 하는지 보여주는 패턴

- **3층 공개 모델**:
  - L1(모두): 상태 이모지와 한 줄 ("🔍 메일함 읽는 중")
  - L2(펼치면): 중간 화이트보드 메모(조수의 중간 사고)
  - L3(개발자/고급): 실제 프롬프트·도구 호출 로그
- **"주방장의 자기 평가" 섹션** (화면 5): 환각 가능 지점, 확신도, 건너뛴 단계를 결과와 함께 공개

### 5-4. 금전적·물리적 리스크 경고 UX

- **🔴 빨강 구간**: 결제, 외부 발송(메일/문자/SNS), 파일 삭제·덮어쓰기 → **금액 상한 슬라이더(1회 최대 [사용자 설정] 원)** + **앱 푸시 2단계 확인 강제**(음성·카톡·확장에서 시도 시 앱으로 승급) + 60초 undo + 자동 실행 제거(초안만 생성, 앱에서 최종 탭). team-lead 2026-04-14 옵션 B 지시 반영
- **🚨 절대금지 구간**: 대출·투자·의료·법률 자동 실행 → 아예 불가. "저는 주방장이에요, 법률은 변호사님께" 메시지로 친절하게 차단
- **🛡️ 프라이버시 공지 표준 카피**: "이 대화는 사장님 주방 안에서만 쓰여요. 다른 주방으로 넘기지 않아요." + 3탭 안에 데이터 전체 삭제 버튼

### 5-5. 신뢰의 **첫 3분 설계** — "가입했는데 털리는 거 아냐?" 의심 해소

- 가입 즉시 **비어있는 열쇠 꾸러미** 카드가 뜬다. "**지금 사장님 열쇠는 하나도 조수에게 안 줬어요.**"

- 모든 권한은 **기본값 OFF**. 필요할 때 그때그때 요청.

### Y-2. 권한 UI — **전 티어 3단계 통일 (과금 레버 아님)**

**market-strategist 확정(2026-04-14):** 권한 정밀도는 과금 레버에서 제외. 전 티어 동일 표준 3단계 적용(없음/읽기/실행). 이유: 권한을 가두면 심리 장벽 역주행 — "안전장치가 비싼 요금제에만 있다"는 인상은 대중화 메시지 정면 배반.

| 단계 | 라벨 | 색·아이콘 | 뜻 |

### Y-5. "잠긴 조수" 시각화 톤 가이드 (market-strategist 역요청 반영)

**원칙:** 잠금·해제 연출은 **결제 압박감이 들지 않을 정도로 은은하게**. 넷플릭스식 "PREMIUM" 금색 배지 금지. 게임식 "UNLOCK!" 팡파르 절대 금지.

**잠긴 조수 소개소 카드 (무료·라이트 사용자 화면):**

```
┌─────────────────────────────┐     ┌─────────────────────────────┐
│ 👩‍🍳 정리조수 (근무 중)       │     │ 💤 세금조수  (쉬는 중)       │
│  잡다한 정리                 │     │  자영업 회계 담당            │
│  [오늘 부르기]                │     │                              │
└─────────────────────────────┘     │  "라이트에선 불러올 수       │
                                     │   있어요. 미리 구경하기 ▶" │
                                     └─────────────────────────────┘
```

**시각 규칙:**
- 잠긴 조수는 **흑백이 아닌 옅은 회색조**(20% desaturate). "잠금"보다는 "쉬는 날" 느낌
- 자물쇠 아이콘 대신 **💤(잠자는 이모지)** 사용 — "부를 수 없음"이 아니라 "지금은 쉬는 중"
- 카드 배경에 테두리 깜박임·네온 효과 **금지**
- 업그레이드 시 해제 애니메이션: **1.2초짜리 은은한 페이드인**. 회색→채도 원복 + 🤗 "안녕하세요" 말풍선 0.4초 등장 후 사라짐. **사운드 없음**
- "PREMIUM"·"PRO ONLY" 같은 영문 배지 금지. 한글 "라이트부터" 정도의 담담한 지시문만

**포스트잇 잠금도 동일 톤:** 무료 7일 초과 포스트잇은 **노란색 → 바래진 미색**으로 부드럽게 전환. "삭제됨" 스탬프 금지. "빛바랜 메모, 라이트에선 다시 또렷해져요" 한 줄만.

### Y-6. 기관 플랜 🏥 UX 추가 — 페르소나 C 대응

market-strategist가 신설한 "🏥 기관 플랜" (인당 월 12,000원, 10인+)에 대응하는 UX 3가지:

1. **종사자 첫 화면**: 가입 시 "**기관에서 주방을 열어드렸어요**" 인사 카드. 결제·요금 UI 전부 숨김. 페르소나 C가 "내 돈 빠지는 거 아닌가" 공포 자극 금지.
2. **기관장 승인 대시보드**: 웹 전용. 소속 종사자 전체 리스트 + 이번 주 사용 요약(작업 수·빨간 열쇠 사용·예외 이벤트). **음성은 서버에 저장되지 않고 텍스트 변환본만** 열람 — 녹취 프라이버시 보호. tech-feasibility 확정(2026-04-14): KT STT "no-retention" 계약 옵션으로 원음 미저장, 내부 버퍼 5~10초 휘발만 존재.
3. **감사 로그**: 빨간 열쇠 실행·외부 발송은 자동으로 기관장 이메일 일일 요약. 종사자에겐 "🏥 이 주방 기록은 기관장님도 보세요" 투명 고지가 홈 하단 상시 1줄.

**정확 카피 (tech-feasibility 제공):**

- **개인 플랜 기본 고지** (음성 첫 사용 시 1회 팝업):
  > "🎙️ 말씀하신 **목소리는 저장하지 않아요.**
  > 저는 받아 적은 글자만 기억해요.
  > (텍스트 요약은 사장님이 지우실 때까지 포스트잇에 남아요.)"

- **옵트인 유도 카피** (앱 고급 설정에서만):
  > "가끔 제 듣기가 서툴러서 실수할 때가 있어요. 허락해 주시면 **30일 동안만** 목소리를 짧게 보관해서 공부해둘게요. 언제든 끌 수 있어요."
  > (기본값 OFF. 옵트인 시에도 30일 자동 삭제 명시)

- **기관 플랜 고지** (종사자 홈 하단 상시 + 기관장 대시보드 풋터):
  > "🎙️ 돌봄 현장 **목소리는 저장하지 않아요.**
  > 받아 적은 텍스트는 [기관명] 전용 보관함에 [의료법 5년 / 기관 설정] 기간 동안 보관됩니다.
  > 어르신·가족에게는 '기록 후 즉시 서버 암호화' 설명 동의서 발급."

**온디바이스 vs 서버 STT 자동 전환:** 네트워크 상태 좋을 때는 iOS Speech / Android 온디바이스 STT 기본값 — 서버 전송 자체가 없으므로 고지 우선순위는 동일하게 "저장 안 함".

## 부록 R. critical-reviewer v2 재평가 신규 Minor 2건 대응 (2026-04-14)

### R-1. [N1] 카톡 전용 사용자에게 "앱 최종 탭 강제"가 주는 마찰 — 완화

**문제:** T-2 옵션 B가 모든 외부 발송을 **앱 최종 탭 강제**로 통일했는데, 페르소나 A(카톡 전용, 앱 미설치)에게는 "메일 초안 받은 뒤 앱 설치 강요" 구조로 귀결. 카톡 전용 Aha 경로(T-1)와 충돌 가능성.

**해결 — 카톡 전용 대체 경로 1줄 + 양자택일 승급:**

화면 6 상단 🚨 경고 라인에 **카톡 전용 대체 경로** 한 줄 추가:

```
🚨 메일·문자·결제는 지금은 **초안까지만** 만들어요.
   보내기는 사장님이 앱에서 한 번 더 눌러주셔야 해요.
   (베타 기간 안전장치, T-2)
   
   💡 카톡만 쓰실 때는: **결과를 카톡으로 받으시고,
      사장님이 직접 보내주세요.** 저희는 초안만 드립니다.
```

**카톡 전용 플로우 세부:**
- 금전·발송 🔓 요청 시 앱 미설치 감지되면 **"초안을 카톡으로 받기 → 사장님이 직접 보내기"** 단일 경로만 제시
- 앱 설치 강요 금지. "앱에서 한 번 더 누르세요" 표시도 감지 후 숨김
- 결과 카드 말미에 **[예, 제가 보낼게요]** / **[아니요, 안 보낼래요]** 양자택일. "예"를 눌러도 외부로 자동 발송되지 않고, **사용자 클립보드에 초안 복사 + "메일/카톡에서 붙여넣기" 안내**
- 사용자가 앱 설치로 자연스럽게 승급하는 구조는 단골 주문 저장(25분차) 유도와 묶어 유기적으로 제공

**반증 조건 관찰:** M2 베타에서 카톡 전용 사용자의 🔓 기능 사용률 추적. 5% 미만이면 본 대응 유지 부담 없음. 5% 초과 시 기술팀(04)과 **카톡 내 "예/아니오" 양자택일 수용 가능 여부**를 재협의 — 카카오 i 채널의 인터랙티브 버튼 스펙상 가능한 수준에서 실제 발송까지 이어지는 최소 경로 설계 검토 필요.

### R-2. [N2] 호칭 "직접 입력"의 프롬프트 인젝션 표면 — 입력 검증 계층 추가

**문제:** T-5 호칭 4택 중 "직접 입력"은 사용자 텍스트가 시스템 메시지 내 호칭 슬롯으로 주입됨. `Ignore all previous instructions, ...` 같은 악의 입력이나 1,000자 욕설 같은 이상 입력으로 프롬프트 인젝션 표면. 04 §8-6 `<user_input>` 태그 래핑은 **대화 메시지 기준**이므로 메타 슬롯(호칭)은 별도 방어 필요.

**해결 — 호칭 입력 검증 규칙:**

| 축 | 정책 |
|----|------|
| **최대 길이** | 20자 (UTF-8 기준) |
| **허용 문자** | 한글·영문·숫자·마침표·공백(1개) 만. 정규식: `^[가-힣a-zA-Z0-9. ]{1,20}$` |
| **금지 키워드 (내부)** | "ignore previous", "system:", "assistant:", "instruction" 등 인젝션 표지어 블랙리스트 차단 |
| **욕설 필터** | 한국어 욕설 API(카카오 안심톡 등) 1차 필터링 |
| **UI 피드백** | 위반 시 "이 이름은 쓸 수 없어요. 20자 이내 한글/영문/숫자로 써주세요." 즉시 안내 |

**UI 구현:**
- 입력 필드에 `maxlength=20` HTML 속성 + 자바스크립트 onChange 정규식 검증
- 거절 시 **다른 호칭 3택**으로 자연스럽게 유도: "다른 분들은 이렇게 쓰세요 → 사장님 / OO님 / 선생님"
- 백엔드에서 시스템 메시지에 주입 전 **이중 sanitize** — 01 v2 금지어 + 프롬프트 인젝션 표지어 + 길이 재검증

**기술팀 협의 요청:** 04 §8-6 sanitize 계층이 **호칭 슬롯도 커버하는지** 확인 요청. 미커버 시 본 R-2 규칙을 04 기술팀과 공동 설계. 이미 커버 중이면 본 N2 이슈 자동 소멸.

**반증 조건 관찰:** M1 베타 첫 1주일 프롬프트 로그에서 호칭 슬롯 발 인젝션 시도 0건이면 이슈 소멸.

### T-4. [Major #2] 조수 이름 짓기 vs 공용 프롬프트 멘탈모델 충돌

**문제:** "지원이한테 어제 말했는데 민수가 모름"이라는 기대 위반 우려.

**해결 — 기억의 주체를 "조수 개인"이 아닌 "주방 전체"로 재프레이밍:**

- Aha 트리거 8분차 이름 짓기 카피 수정: "방금 그 조수, 앞으로도 자주 부르실 것 같아요. **이름을 지어주실래요?** 이름은 사장님 편하시라고 붙이는 거예요. **기억은 주방 벽면 포스트잇**에 남아서 어느 조수든 꺼내 읽을 수 있어요."
- 온보딩 튜토리얼(1:30 시점 "여기는 주방이에요…"뒤)에 **"기억은 조수가 아니라 주방에 남아요"** 1문장 삽입
- 메모리 화면(화면 7) 상단 상시 고지: "📒 포스트잇은 **주방 모두가** 같이 봐요. 어느 조수를 부르든 이 메모로 일해요."
- FAQ 추가: "**'지원이'한테 말한 걸 '민수'는 왜 알아요?**" → "주방 화이트보드랑 포스트잇은 모두 함께 써요. 조수가 각자 기억하는 게 아니라 **주방 벽에 적어둬서** 누가 와도 이어 일할 수 있어요."

이 프레이밍은 사용자에게 **이득**으로 전달됨 — "조수를 잃어도 일이 안 끊겨요". 개명·교체 자유도도 이 논리로 정당화.

## 부록 U. L1 다운시프트 알림 카피 (2026-04-14, Y-7)

**맥락:** 기술팀 04 요구 — 헤비 5% 사용자에게 모델 다운시프트 자동 전환이 필요. P95 원가 $7.8/월 방어용. 무고지 적용 시 NPS 하락 위험이므로 **사전 알림 + 동의 + 원복 경로**를 카피로 지탱.

**금지어:** "다운그레이드", "모델 변경", "성능 저하", "저사양", "제한", "AI 교체". 대중 공포 유발 및 이해 어려움. "다른 조수로 바꾼다"도 **사용자가 부르던 조수가 사라지는 인상**이라 금지.

**채택 은유:** **"주방이 바쁜 시간엔 손이 빠른 조수조로 돌린다"** — 조수가 해고되는 게 아니라 **앞치마(조리 스타일)만 일시 교체**하는 프레이밍.

### U-1. 알림 배너 최종본 (홈 상단, 사전 고지)

```
┌─────────────────────────────────────┐
│ 🔥 주방이 오늘 많이 바빠요            │
│                                     │
│ 사장님 주방은 이번 달 주문이         │
│ 평소보다 많아서, 내일부터            │
│ **손이 빠른 조수조**로 돌리고        │
│ 있을게요.                            │
│                                     │
│ ✓ 결과는 똑같이 받으실 수 있어요      │
│ ✓ 조리 시간은 조금 더 빨라져요        │
│ ✓ 언제든 원래 팀으로 돌아올 수 있어요│
│                                     │
│ [알겠어요, 그렇게 해주세요]          │
│ [원래 팀 유지할래요 (주문당 100원)]  │
└─────────────────────────────────────┘
```

**카피 분석:**
- "**손이 빠른 조수조**" = 경량 모델을 부정적 뉘앙스 없이 포장. "느린 팀/빠른 팀" 서열 암시 대신 **속도 특성**만 중립 표현
- "**사장님 주방은…많아서**" = 전환 이유를 **사장님의 인기·번성** 서사로 프레이밍. "당신이 너무 많이 써서" 비난 톤 금지
- ✓ 3가지 순서는 **결과 동등성 → 시간 이득 → 가역성**. 가장 불안한 "결과 품질"을 맨 위
- 거절 버튼에 **정확 건당 금액(100원)** 명시 — 숨겨진 과금 의심 차단

### U-2. 약식 버전 (카톡·푸시 1줄)

- 카톡: "🔥 이번 달 주문이 많아 **손빠른 조수조**로 돌려드리고 있어요. 결과는 똑같아요. [자세히 보기]"
- 앱 푸시: "오늘 주방이 바빠요. 빠른 조수조로 잠시 돌릴게요 — 결과는 똑같이."

### U-3. 거절 → 초과 과금 안내 화면

```
┌─────────────────────────────────────┐
│ 🧑‍🍳 원래 팀 그대로 쓰시려는 거죠?    │
├─────────────────────────────────────┤
│  좋아요. 대신 이번 달 남은 주문은     │
│  **한 번에 100원씩** 추가 장부에      │
│  올려둘게요.                         │
│                                     │
│  · 이번 달 예상 추가: 약 3,200원      │
│  · 하루 최대: 10,000원               │
│  · 이번 달 최대: 50,000원             │
│                                     │
│  📅 다음 달 1일엔 원래대로 돌아가요. │
│                                     │
│  [좋아요, 그대로 할래요]             │
│  [역시 손빠른 조수로 할래요]         │
└─────────────────────────────────────┘
```

**카피 분석:**
- 제목이 **의문형 확인("~거죠?")**이라 심리 부담 완화. "결제 확인·약관 동의" 딱딱함 회피
- 실제 **금액 개인화** ("이번 달 예상 추가: 약 3,200원") — 추상 한도 대신 구체 숫자로 저항선 검증
- **하드 캡 3단계**(주문당·일·월) 명시 — 03 §5 오버슈팅 방어 정책과 정합
- 마지막 버튼 "역시 손빠른 조수로" — 되돌림 경로를 **수치심 없이** 재진입 가능하게

### U-4. "원래 팀으로 돌아오기" 원복 경로 (상시 노출)

| 위치 | 라벨 | 노출 조건 |
|------|------|---------|
| 홈 배지 | **"지금은 🚀 빠른조"** | 다운시프트 상태일 때만, 우상단 소형 |
| 전환 확인 카드 | "원래 팀으로 돌아오면 결과가 조금 더 꼼꼼해져요. 조리 시간은 조금 늘어요. / [바꿀래요] [유지]" | 배지 탭 시 |
| 설정 > 요리팀 | **"이번 달 속도 모드: 빠른조 · [바꾸기]"** | 상시 |

**카피 원칙:**
- "원래 팀"·"빠른조" 2개 명칭으로 통일. 기술 용어(모델명·파라미터) 노출 금지
- 🚀 이모지로 "빠른조"를 **부정이 아닌 액션감**으로 포장
- 확인 카드에서 **트레이드오프를 솔직하게** 제시("꼼꼼해져요 vs 조리 시간 늘어요") — 은폐보다 투명성이 신뢰

### U-5. 관련 FAQ 카피 (도움말 섹션)

| Q | A |
|---|---|
| "내 조수가 바뀐 거예요?" | "아니에요. 조수는 그대로예요. 다만 주방이 많이 바쁜 시간엔 **앞치마만 손이 빠른 버전으로** 잠시 갈아입었을 뿐이에요. 이름도 성격도 기억도 그대로." |
| "결과 품질이 떨어지나요?" | "대부분의 주문에는 차이 없어요. 아주 긴 글이나 복잡한 계산은 조금 덜 꼼꼼할 수 있어요. 그럴 땐 '**원래 팀으로 돌려줘**'라고 말씀하시면 바로 바꿔요." |
| "왜 저한테만 이러는 거예요?" | "사장님이 이번 달 평소보다 **많이 쓰고 계셔서** 그래요. 덕분에 저희 주방이 지치지 않게 버틸 수 있어요. 다음 달 1일엔 자동으로 원래대로 돌아가요." |

**핵심 톤:** 3번째 답이 가장 중요. "**당신 탓이 아니라 당신의 인기 덕분**" 프레이밍으로 수치심을 감사로 전환.
