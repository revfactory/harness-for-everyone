# 사용자 요청

**원문:** "하네스를 일반 대중에게 쉽게 전달할 방법 찾아줘"

**요청 시각:** 2026-04-14

## Phase 1 분석

| 항목 | 값 | 근거 |
|------|-----|------|
| 타겟 페르소나 | 미지정 → 기본 가정 적용 | 사용자 언급 없음 |
| 지역/시장 | 미지정 → 한국 기반, 글로벌 확장 고려 | 사용자 언급 없음 |
| 예산 제약 | 미지정 → 무료~월 2만원 구간 | 기본 가정 |
| 톤 요구 | 미지정 → "쉽게" 전달에 중점 → 대중 친화적, 은유 중심 | 입력에 "쉽게"가 명시 |
| 강조 축 | **개념 번역(전달)**을 최우선 — "전달할 방법"이라는 표현은 UX/마케팅/교육을 포괄 | 입력 문맥 |

## 가정 (기본값)

- **페르소나:** 한국 거주, 비개발 일반인(30~50대 지식 노동자 중심), 스마트폰/PC 사용 능숙, AI 챗봇 경험은 있으나 에이전트/하네스 개념은 생소
- **가격 상한:** 무료 티어 포함, 유료는 월 2만원 이하
- **기기 제약:** 터미널 없음, 웹/모바일/메신저/음성 기반
- **기대 산출 톤:** 제안서 형식이되 일반인이 읽어도 이해 가능한 은유·스토리 포함
- **성공 기준:** (a) 하네스 개념이 3가지 이상 일상 은유로 설명됨 (b) 비개발자가 30분 안에 체험 가능한 경로 제시 (c) 월 2만원 이하로 핵심 가치 재현 가능한 기술 스택 제시 (d) 최소 3개의 실패 시나리오에 대한 대응책 포함

## 팀에 전달할 지침

1. **"전달"의 4면:** 개념(은유) · 경험(UX) · 가격(BM) · 기술(실현) — 네 축 모두 필수. 한 축만 강한 제안은 실패로 간주.
2. **일반인 친화성 테스트:** 모든 산출물은 "30대 비개발자 지인에게 카톡으로 보냈을 때 이해되는가?"를 자체 검증 통과해야 함.
3. **구체성:** "교육이 필요하다", "UX를 개선해야 한다" 같은 추상어 금지. 실제 은유·화면 플로우·가격표·기술 스택·실패 시나리오까지 명시.
4. **상충 허용:** 팀원 간 의견이 합의되지 않으면 양안 병기 후 synthesis-lead 판단.
